Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Authentication

APIs for managing user authentication.

Register a new user

Example request:
curl --request POST \
    "http://localhost/api/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"name\": \"John Doe\",
    \"email\": \"john@example.com\",
    \"password\": \"password123\",
    \"confirm_password\": \"eius\"
}"

const url = new URL(
    "http://localhost/api/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "name": "John Doe",
    "email": "john@example.com",
    "password": "password123",
    "confirm_password": "eius"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (201):


{
    "user": {
        "id": 1,
        "name": "John Doe",
        "email": "john@example.com"
    },
    "token": "1|abcdef123456..."
}

        "The name field is required."

    "email": [
        "The email field is required."
    ],

Task Management

APIs for managing tasks.

List all tasks with filtering, sorting, and pagination.

Example request:
curl --request GET \
    --get "http://localhost/api/tasks?showArchived=&priority=High&due_date_from=2023-11-01&due_date_to=2023-11-30&completed_date_from=2023-11-01&completed_date_to=2023-11-30&archived_date_from=2023-11-01&archived_date_to=2023-11-30&search=project&sort_by=due_date&sort_order=asc&page=2" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost/api/tasks"
);

const params = {
    "showArchived": "0",
    "priority": "High",
    "due_date_from": "2023-11-01",
    "due_date_to": "2023-11-30",
    "completed_date_from": "2023-11-01",
    "completed_date_to": "2023-11-30",
    "archived_date_from": "2023-11-01",
    "archived_date_to": "2023-11-30",
    "search": "project",
    "sort_by": "due_date",
    "sort_order": "asc",
    "page": "2",
};
Object.keys(params)
    .forEach(key => url.searchParams.append(key, params[key]));

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());

GET api/tasks

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Query Parameters

showArchived   boolean  optional  

Optional. If true, includes archived tasks. Default: false. Example: false

priority   string  optional  

Optional. Filter by priority level (Urgent, High, Normal, Low). Example: High

due_date_from   string  optional  

date Optional. Filter tasks due from this date (inclusive). Example: 2023-11-01

due_date_to   string  optional  

date Optional. Filter tasks due up to this date (inclusive). Example: 2023-11-30

completed_date_from   string  optional  

date Optional. Filter tasks completed from this date (inclusive). Example: 2023-11-01

completed_date_to   string  optional  

date Optional. Filter tasks completed up to this date (inclusive). Example: 2023-11-30

archived_date_from   string  optional  

date Optional. Filter tasks archived from this date (inclusive). Example: 2023-11-01

archived_date_to   string  optional  

date Optional. Filter tasks archived up to this date (inclusive). Example: 2023-11-30

search   string  optional  

Optional. Search for tasks by title or description. Example: project

sort_by   string  optional  

Optional. Sort by field (title, description, due_date, created_at, completed_at, priority). Example: due_date

sort_order   string  optional  

Optional. Sort order (asc, desc). Default: asc. Example: asc

page   integer  optional  

Optional. Page number for pagination. Default: 1. Example: 2
